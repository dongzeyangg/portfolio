Installation, configuration et virtualhost sur Debian/Ubuntu.

Apache 2.4 — Debian 12 / Ubuntu 22.04+
01

Installer Apache2Installation

Terminal # Mettre à jour les paquets #apt update # Installer Apache2 #apt install apache2 -y
✅

Apache démarre automatiquement après l'installation. Ouvrez http://localhost dans un navigateur pour voir la page de test.

02

Gérer le serviceService Management

Terminal # Démarrer Apache #systemctl start apache2 # Arrêter Apache #systemctl stop apache2 # Redémarrer Apache #systemctl restart apache2 # Activer au démarrage du système #systemctl enable apache2 # Vérifier le statut #systemctl status apache2
03

Pare-feu (UFW)Firewall

Terminal # Autoriser HTTP (port 80) #ufw allow 'Apache' # Autoriser HTTP + HTTPS (ports 80 et 443) #ufw allow 'Apache Full' # Vérifier les règles actives #ufw status
💡

Si UFW n'est pas actif, activez-le d'abord avec ufw enable avant d'ajouter des règles.

04

Structure des fichiersFile Structure

  • 📁/etc/apache2/ — Dossier de configuration principal
  • 📄/etc/apache2/apache2.conf — Configuration globale
  • 📁/etc/apache2/sites-available/ — VirtualHosts disponibles
  • 📁/etc/apache2/sites-enabled/ — VirtualHosts actifs
  • 📁/var/www/html/ — Racine web par défaut
  • 📋/var/log/apache2/ — Journaux d'accès et d'erreurs
VirtualHost — Héberger plusieurs sites sur un même serveur
01

Créer le dossier du siteWeb Root

Terminal # Créer le dossier racine du site #mkdir -p /var/www/monsite.fr/html # Attribuer les permissions à l'utilisateur courant #chown -R $USER:$USER /var/www/monsite.fr # Créer une page d'accueil de test $echo "<h1>Bienvenue sur monsite.fr</h1>" > /var/www/monsite.fr/html/index.html
02

Créer le fichier VirtualHostConfig File

Terminal #nano /etc/apache2/sites-available/monsite.fr.conf
# Contenu du fichier :
<VirtualHost *:80>
    ServerName     monsite.fr
    ServerAlias    www.monsite.fr
    DocumentRoot   /var/www/monsite.fr/html
    ErrorLog       ${APACHE_LOG_DIR}/monsite-error.log
    CustomLog      ${APACHE_LOG_DIR}/monsite-access.log combined
</VirtualHost>
💡

Remplacez monsite.fr par votre nom de domaine réel ou une IP locale pour les tests.

03

Activer le siteEnable Site

Terminal # Activer le VirtualHost #a2ensite monsite.fr.conf # Désactiver le site par défaut (optionnel) #a2dissite 000-default.conf # Vérifier la configuration Apache #apache2ctl configtest # Recharger Apache #systemctl reload apache2
✅

Votre site est actif. Testez-le en tapant http://monsite.fr dans un navigateur (ou en ajoutant l'IP dans votre fichier /etc/hosts pour un test local).

HTTPS — Certificat SSL gratuit avec Let's Encrypt
01

Installer CertbotLet's Encrypt

Terminal # Installer Certbot et le plugin Apache #apt install certbot python3-certbot-apache -y
⚠️

Votre nom de domaine doit pointer vers votre serveur (enregistrement DNS A configuré) avant de générer le certificat.

02

Générer le certificat SSLCertificate

Terminal # Générer et installer le certificat automatiquement #certbot --apache -d monsite.fr -d www.monsite.fr
  1. 1
    Entrez votre adresse e-mail pour les notifications d'expiration.
  2. 2
    Acceptez les conditions d'utilisation de Let's Encrypt.
  3. 3
    Choisissez de rediriger HTTP vers HTTPS (option recommandée).
✅

Certbot configure Apache et active HTTPS automatiquement. Votre site est maintenant accessible en https://monsite.fr.

03

Renouvellement automatiqueAuto-Renewal

Terminal # Tester le renouvellement automatique #certbot renew --dry-run # Le renouvellement est automatique via systemd timer #systemctl status certbot.timer
💡

Les certificats Let's Encrypt sont valables 90 jours. Certbot renouvelle automatiquement avant expiration grâce à un timer systemd.
