02

Installer Apache2 & PHPWeb Server

Terminal # Mettre à jour le système #apt update && apt upgrade -y # Installer Apache2 #apt install apache2 -y # Installer PHP 8.2 et toutes les extensions requises par GLPI #apt install php8.2 php8.2-mysql php8.2-xml php8.2-curl \   php8.2-gd php8.2-mbstring php8.2-intl php8.2-zip \   php8.2-bz2 php8.2-ldap php8.2-opcache \   libapache2-mod-php8.2 -y
03

Installer MariaDBDatabase

Terminal # Installer MariaDB #apt install mariadb-server -y # Sécuriser l'installation (suivre les étapes interactives) #mysql_secure_installation
⚠️

Lors de mysql_secure_installation : définissez un mot de passe root fort, supprimez les utilisateurs anonymes et désactivez l'accès root à distance.

04

Créer la base de données GLPIDatabase Setup

Terminal # Se connecter à MariaDB #mysql -u root -p
-- Créer la base de données
CREATE DATABASE glpi CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- Créer l'utilisateur GLPI
CREATE USER 'glpi'@'localhost' IDENTIFIED BY 'MotDePasseFort!';

-- Accorder les droits
GRANT ALL PRIVILEGES ON glpi.* TO 'glpi'@'localhost';

FLUSH PRIVILEGES;
EXIT;
GLPI 10.x — Téléchargement & déploiement
01

Télécharger GLPIDownload

Terminal # Aller dans le dossier temporaire #cd /tmp # Télécharger la dernière version de GLPI #wget https://github.com/glpi-project/glpi/releases/download/10.0.16/glpi-10.0.16.tgz # Extraire l'archive #tar -xzvf glpi-10.0.16.tgz -C /var/www/html/
💡

Vérifiez la dernière version disponible sur github.com/glpi-project/glpi/releases et adaptez le numéro de version dans la commande.

02

Permissions & configuration ApachePermissions

Terminal # Attribuer les droits à Apache #chown -R www-data:www-data /var/www/html/glpi #chmod -R 755 /var/www/html/glpi # Créer le VirtualHost Apache pour GLPI #nano /etc/apache2/sites-available/glpi.conf
<VirtualHost *:80>
  ServerName    glpi.mondomaine.fr
  DocumentRoot /var/www/html/glpi/public
  <Directory /var/www/html/glpi/public>
    Options -Indexes
    AllowOverride All
    Require all granted
  </Directory>
</VirtualHost>
Terminal # Activer le module rewrite et le site #a2enmod rewrite #a2ensite glpi.conf #systemctl restart apache2
03

Installation via le navigateurWeb Installer

  1. 1
    Ouvrez http://glpi.mondomaine.fr dans un navigateur.
  2. 2
    Choisissez la langue et acceptez la licence GPL.
  3. 3
    Cliquez sur Installer (pas Mettre à jour).
  4. 4
    Entrez les informations de la base de données : hôte localhost, utilisateur glpi, mot de passe et base glpi.
  5. 5
    Laissez l'installation se terminer et cliquez sur Continuer.
✅

GLPI est installé ! Connectez-vous avec glpi / glpi (admin par défaut). Changez ce mot de passe immédiatement.

Post-installation — Sécurité & premiers réglages
01

Supprimer le fichier d'installationSecurity

Terminal # OBLIGATOIRE — supprimer install.php après installation #rm /var/www/html/glpi/install/install.php
🔐

Ne pas supprimer ce fichier expose votre installation GLPI à une réinstallation non autorisée par n'importe qui.

02

Changer les mots de passe par défautDefault Accounts

  • 👤glpi / glpi — compte administrateur principal
  • 👤tech / tech — compte technicien
  • 👤normal / normal — compte utilisateur standard
  • 👤post-only / postonly — compte en libre-service
⚠️

Changez tous ces mots de passe dès la première connexion via Administration → Utilisateurs.

03

Configurer les tâches automatiquesCron

Terminal # Ajouter le cron GLPI (exécution toutes les minutes) #crontab -u www-data -e
# Ajouter cette ligne dans le crontab :
* * * * * /usr/bin/php /var/www/html/glpi/front/cron.php &>/dev/null
💡

Le cron est indispensable pour les notifications e-mail, la clôture automatique des tickets et les autres tâches planifiées de GLPI.

04

Premiers réglages à faireQuick Setup

  1. 1
    Configuration → Générale — renseignez le nom, l'URL et le fuseau horaire de votre instance.
  2. 2
    Configuration → Notifications — configurez le serveur SMTP pour les e-mails.
  3. 3
    Administration → Entités — créez votre structure organisationnelle (entreprise, sites…).
  4. 4
    Administration → Utilisateurs — créez les comptes ou configurez l'annuaire LDAP/AD.
  5. 5
    Assistance → Catégories — créez les catégories de tickets (matériel, logiciel, réseau…).
🚀

Pour connecter GLPI à votre Active Directory, allez dans Configuration → Annuaires LDAP et renseignez l'IP de votre contrôleur de domaine.
