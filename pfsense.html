pfSense CE 2.7.x — FreeBSD · Open Source
01

Prérequis matérielsHardware

  • 🖥️CPU : 64-bit, 1 GHz minimum (2+ GHz recommandé)
  • 💾RAM : 1 GB minimum — 2 GB+ recommandés
  • 💿Disque : 8 GB minimum (SSD recommandé)
  • 🌐Cartes réseau : 2 minimum — une WAN (internet), une LAN (réseau local)
  • 🔌Clé USB : 1 GB minimum pour l'image ISO
⚠️

pfSense nécessite au minimum 2 interfaces réseau. Sur une VM, créez deux cartes réseau avant de démarrer l'installation.

02

Télécharger & créer la clé USBDownload

  1. 1
    Téléchargez l'image ISO sur netgate.com/pfsense-plus-software (CE = gratuit).
  2. 2
    Choisissez AMD64, format DVD Image (ISO).
  3. 3
    Créez la clé USB avec Rufus : schéma GPT, système cible UEFI.
💡

pfSense peut aussi s'installer directement sur une machine virtuelle (VMware, Proxmox, VirtualBox) en montant l'ISO comme lecteur CD.

03

Installer pfSenseSetup

  1. 1
    Démarrez sur la clé USB. À l'écran de démarrage, appuyez sur Entrée pour lancer l'installateur.
  2. 2
    Acceptez le contrat de licence, puis choisissez Install pfSense.
  3. 3
    Keymap : sélectionnez French (accent keys) pour un clavier AZERTY.
  4. 4
    Partitionnement : choisissez Auto (ZFS) recommandé, ou Auto (UFS) pour les petites machines.
  5. 5
    Sélectionnez le disque de destination et confirmez. L'installation démarre (~5 min).
  6. 6
    À la fin, choisissez Reboot et retirez la clé USB.
✅

pfSense redémarre et affiche une console texte avec le menu de configuration. C'est normal — l'interface web viendra ensuite.

02

Assigner les interfaces (console)Interface Assignment

  1. 1
    Au menu console, choisissez l'option 1 — Assign Interfaces.
  2. 2
    Répondez n à la question sur les VLANs (sauf si vous en avez besoin).
  3. 3
    Assignez l'interface WAN : entrez le nom de la carte connectée à internet (ex : em0).
  4. 4
    Assignez l'interface LAN : entrez le nom de la seconde carte (ex : em1).
  5. 5
    Confirmez avec y. pfSense applique la configuration.
💡

pfSense affiche les adresses MAC de chaque carte. Comparez-les avec celles visibles dans votre BIOS ou hyperviseur pour identifier quelle carte est quelle interface.

03

Configurer l'IP du LAN (console)LAN IP

  1. 1
    Au menu console, choisissez 2 — Set interface(s) IP address.
  2. 2
    Choisissez 2 — LAN.
  3. 3
    Entrez l'IP LAN souhaitée, ex : 192.168.1.1 avec masque 24.
  4. 4
    Activez le serveur DHCP sur le LAN si vous voulez que pfSense distribue les IP automatiquement.
✅

L'interface web est maintenant accessible depuis le LAN sur https://192.168.1.1 avec admin / pfsense (à changer immédiatement).

04

Configuration initiale via WebGUIWeb Interface

  1. 1
    Depuis un PC du LAN, ouvrez https://192.168.1.1 et acceptez l'avertissement de certificat.
  2. 2
    Connectez-vous avec admin / pfsense et suivez l'assistant de configuration.
  3. 3
    Renseignez le nom d'hôte, le domaine et les serveurs DNS (ex : 8.8.8.8).
  4. 4
    Vérifiez la configuration WAN — DHCP si votre FAI l'attribue automatiquement, sinon IP statique.
  5. 5
    Définissez un nouveau mot de passe admin et terminez l'assistant.
🔐

Changez le mot de passe admin / pfsense immédiatement. Ce mot de passe par défaut est public et connu de tous.

Firewall — Règles, NAT & redirection de ports
01

Comprendre les règlesFirewall Rules

Les règles pfSense sont évaluées de haut en bas. La première règle qui correspond est appliquée.

Pass (Autoriser)

Laisse passer le trafic correspondant à la règle.

Block (Bloquer)

Bloque silencieusement le trafic sans notifier l'émetteur.

NAT / Port Forward

Redirige un port entrant vers une machine du LAN.

Reject

Bloque et notifie l'émetteur (TCP RST / ICMP unreachable).

💡

Par défaut, le LAN peut tout sortir (règle Allow LAN to any) et le WAN bloque tout entrant. C'est le comportement le plus sûr pour démarrer.

02

Créer une règle de pare-feuAdd Rule

  1. 1
    Allez dans Firewall → Rules → LAN (ou WAN).
  2. 2
    Cliquez sur Add ↑ pour ajouter une règle en haut de la liste.
  3. 3
    Choisissez l'action (Pass / Block), le protocole (TCP, UDP, ICMP…).
  4. 4
    Définissez la source et la destination (IP, réseau, alias ou "any").
  5. 5
    Ajoutez une description claire, puis cliquez sur Save et Apply Changes.
⚠️

Cliquez toujours sur Apply Changes après avoir sauvegardé une règle, sinon elle ne sera pas active.

03

Redirection de ports (NAT)Port Forwarding

  1. 1
    Allez dans Firewall → NAT → Port Forward.
  2. 2
    Cliquez sur Add. Interface : WAN.
  3. 3
    Protocol : TCP. Destination port range : entrez le port à ouvrir (ex : 80 pour HTTP).
  4. 4
    Redirect target IP : IP locale de votre serveur (ex : 192.168.1.10).
  5. 5
    Redirect target port : port de destination sur le serveur. Sauvegardez et appliquez.
✅

pfSense crée automatiquement la règle firewall associée au NAT. Votre service est accessible depuis internet.

Fonctionnalités avancées — DHCP · DNS · VPN · Packages
01

Serveur DHCPDHCP Server

  1. 1
    Allez dans Services → DHCP Server → LAN.
  2. 2
    Cochez Enable DHCP server on LAN interface.
  3. 3
    Définissez la plage d'adresses (ex : 192.168.1.100 → 192.168.1.200).
  4. 4
    Renseignez les DNS à distribuer et sauvegardez.
💡

Vous pouvez attribuer des baux statiques (IP fixe par adresse MAC) dans la section DHCP Static Mappings en bas de la même page.

02

DNS ResolverDNS

  1. 1
    Allez dans Services → DNS Resolver.
  2. 2
    Vérifiez que le résolveur est activé et que l'interface LAN est bien sélectionnée.
  3. 3
    Activez DNSSEC pour sécuriser les requêtes DNS.
  4. 4
    Optionnel : activez DNS over TLS pour chiffrer les requêtes vers les serveurs amont.
03

VPN OpenVPNRemote Access

  1. 1
    Allez dans VPN → OpenVPN → Wizards pour un assistant guidé.
  2. 2
    Créez une autorité de certification (CA) interne si vous n'en avez pas.
  3. 3
    Choisissez le port (1194 UDP par défaut) et le réseau tunnel (ex : 10.8.0.0/24).
  4. 4
    L'assistant crée automatiquement les règles firewall et NAT nécessaires.
  5. 5
    Exportez le profil client via VPN → OpenVPN → Client Export (installer le package si absent).
🔐

Le VPN permet à vos utilisateurs distants de se connecter au réseau interne de façon sécurisée, comme s'ils étaient sur place.

04

Packages utilesPackages

  • 📦pfBlockerNG — Blocage de publicités et domaines malveillants (DNS sinkhole).
  • 📦Snort / Suricata — Système de détection d'intrusion (IDS/IPS).
  • 📦ntopng — Surveillance du trafic réseau en temps réel.
  • 📦Squid — Proxy cache HTTP pour filtrer et accélérer la navigation.
  • 📦acme — Certificats SSL Let's Encrypt pour la WebGUI.
💡

Installez les packages via System → Package Manager → Available Packages directement depuis la WebGUI.
